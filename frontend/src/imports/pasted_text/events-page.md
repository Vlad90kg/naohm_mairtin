Design a clean, modern Events page for a GAA club website (Naomh Mairtin GFC), using a card-based layout inspired by a mobile app UI.

Style:
- Primary color: deep blue (#1E3A8A or similar)
- White/light grey cards
- Minimal gold accents for highlights
- Rounded corners, soft shadows
- Clean typography, strong hierarchy
- Consistent spacing and grid

---

GLOBAL HEADER:

1. Tier 1 Sponsor Strip (top):
- Horizontal bar with 3–5 sponsor logos
- Clickable logos
- Minimal height

2. Navigation Bar (sticky):
- Logo left
- Menu:
  Home, About (dropdown), Teams (dropdown), Events (active), Lotto, Shop, Membership, Sponsors, Contact, Child Safety

---

PAGE STRUCTURE:

3. Page Header Section:
- Blue background banner
- Title: “Events”
- Short subtitle:
  “Stay up to date with upcoming fixtures and club events”

---

4. Upcoming Events Section (PRIMARY):

- Title: “Upcoming Events”

- Display events as cards in a grid (2–3 per row)

Each event card includes:
- Event title
- Date
- Time (optional)
- Location
- Optional small image/icon
- CTA button: “View Details” or “More Info”

Important:
- This section is the main focus of the page
- Use clear spacing and visual hierarchy

---

5. Past Events Section (SECONDARY):

- Title: “Past Events”

- Display past events in a lighter or more compact card style
- Reduce visual emphasis (e.g. faded style or smaller cards)

Purpose:
- Archive only
- Not primary focus

---

6. Optional Filter / Category Bar:

- Simple filters (if included):
  - All
  - Matches
  - Club Events
  - Training

Keep it simple:
- No complex UI
- Optional for wireframe

---

7. Call to Action (Optional):

- Short message:
  “Get involved in upcoming events”
- Button: “Membership Info”

---

8. Footer:

- Address + Eircode
- Email
- Navigation links
- Social icons
- Clean and compact

---

FUNCTIONAL REQUIREMENTS:

- Use consistent card-based layout
- Prioritise upcoming events over past
- Avoid long text descriptions
- Keep layout simple and scannable
- Do not include login functionality
- Do not overload with unnecessary features
- Ensure responsive layout (cards stack on mobile)
- Match style of homepage and mobile app
